# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a React application for "Gadki" (an educational campaign site), auto-generated by Anima design-to-code platform. The project uses React 18, React Router, Vite, and Tailwind CSS for a Polish/Ukrainian language educational website.

## Development Commands

```bash
# Install dependencies
npm install

# Run development server (http://localhost:5173/)
npm run dev

# Build for production
npm run build
```

## Architecture

### Routing Structure

The app uses React Router v6 with the following route patterns:

- `/` - Home page (GadkiStronaGwna)
- `/menu` - Menu overlay (GadkiMenuGwne)
- `/o-kampanii` - About campaign
- `/dla-dzieci` - For children
- `/dla-rodzicow-i-opiekunow` - For parents and guardians
- `/dla-edukatorow` - For educators
- `/faq` - FAQ page
- `/logowanie` - Login/Registration
- `/konto` - User account

All routes are defined in `src/App.jsx`.

### Directory Organization

The codebase has an unusual structure due to Anima's auto-generation:

```
src/
├── screens/              # Root-level screens (e.g., GadkiStronaGwna home page)
│   └── GadkiStronaGwna/
│       └── sections/     # Screen-specific section components
├── routes/               # Route-specific screens
│   ├── GadkiMenuGwne/
│   │   └── screens/
│   ├── GadkiDlaDzieci/
│   │   └── screens/
│   ├── GadkiDlaEdukatorw/
│   │   └── screens/
│   └── sections/         # SHARED section components used across routes
│       └── screens/
│           └── sections/ # Reusable UI sections (Footer, Newsletter, etc.)
└── components/           # Global reusable components (Header)
```

**Key Pattern**: `src/routes/sections/screens/sections/` contains SHARED section components like:
- `Footer.jsx` - Site footer (used across multiple pages)
- `Newsletter.jsx` - Newsletter signup section
- `BoxesWrapper.jsx`, `Boxes.jsx` - Content box components
- `MoreQuestions.jsx` - Questions section
- `Frame.jsx`, `FrameWrapper.jsx` - Layout frames

These differ from screen-specific sections in `src/screens/GadkiStronaGwna/sections/`, which are local to that screen.

### Navigation Flow

- The `.screen-graph.json` file maps the navigation graph between screens
- Navigation uses React Router's `<Link>` components
- The `Header` component (in `src/components/`) provides logo → home and hamburger → menu navigation
- The menu overlay (`GadkiMenuGwne`) provides main navigation to all sections

## Development Notes

### Working with Anima-Generated Code

This codebase was auto-generated from design files, which means:

1. **Component naming**: Uses Polish characters and descriptive names (e.g., `GadkiDlaRodzicwI`)
2. **Styling patterns**: Heavy use of absolute positioning and fixed pixel values
3. **Assets**: Images referenced from `c.animaapp.com` CDN
4. **Layout**: Screens have fixed dimensions (e.g., `min-w-[1728px]`, `h-[9567px]`)

### Making Changes

When modifying components:
- Screen components are in `routes/[RouteName]/screens/`
- Shared sections are in `routes/sections/screens/sections/`
- The root screen is in `screens/GadkiStronaGwna/`
- Update navigation in `src/App.jsx` when adding routes
- Maintain the existing absolute positioning patterns for consistency

### Component Reuse

Before creating a new section component, check `src/routes/sections/screens/sections/` for existing reusable components like Footer, Newsletter, Boxes, etc.

## Figma Integration

This project uses the `claude-code-figma` CLI tool to extract design information from Figma.

### Converting Figma designs to code

When a Figma link is provided, use the following steps:

1. Extract the Figma metadata:
   ```bash
   claude-code-figma extract <figma-url> --optimize
   ```

2. Based on the metadata, implement the component using:
   - The project's existing Tailwind CSS theme system (see below)
   - The component structure from the Figma design
   - The Tailwind CSS classes provided in the `tailwindClasses` section
   - The naming conventions and directory structure used in the codebase
   - React 18 functional components with hooks

   If the metadata contains `tailwindConfig` suggestions, evaluate whether to add custom colors or other values to `tailwind.config.js`.

### Design System Information

When implementing components from Figma designs, follow these project-specific conventions:

#### Color System

Custom color palette defined in `tailwind.config.js`:

**Beige shades:**
- `beige-100` through `beige-500` - Neutral backgrounds and accents

**Brand colors:**
- `gadkidarkblue` - Primary dark blue
- `gadkidarkgreen` - Primary dark green
- `gadkidarkred` - Primary dark red
- `gadkiorange` - Primary orange
- `gadkiyellow` - Primary yellow
- `fddsraspberry` - Raspberry accent

Use these colors via Tailwind classes: `bg-gadkidarkblue`, `text-gadkiyellow`, etc.

#### Typography

**Fonts:**
- `font-lato` - Lato (primary sans-serif for body text)
- `font-happy-season` - Happy Season (display font for headings)

**Usage:**
- Apply fonts via Tailwind classes: `font-lato`, `font-happy-season`
- Use standard Tailwind text sizing: `text-sm`, `text-base`, `text-lg`, `text-xl`, `text-2xl`, etc.
- Polish characters are supported

#### Spacing & Layout

- Use Tailwind spacing utilities: `m-4`, `p-6`, `gap-8`, etc.
- Fixed dimensions are common (Anima pattern): `min-w-[1728px]`, `h-[9567px]`
- Absolute positioning is heavily used: `absolute`, `relative`, `top-[40px]`, `left-[200px]`
- Maintain existing absolute positioning patterns for consistency with Anima-generated screens

#### Component Structure

**For screen components:**
- Placement: `src/routes/[RouteName]/screens/`
- Example: `src/routes/GadkiDlaDzieci/screens/GadkiDlaDzieci.jsx`

**For shared section components:**
- Placement: `src/routes/sections/screens/sections/`
- Examples: `Footer.jsx`, `Newsletter.jsx`, `BoxesWrapper.jsx`

**For global components:**
- Placement: `src/components/`
- Example: `Header.jsx`

**Component file structure:**
- Single `.jsx` file per component
- Export default component
- Import React and necessary dependencies at top
- Use functional components with hooks

#### Naming Conventions

- **Component files**: PascalCase with Polish characters allowed (e.g., `GadkiDlaRodzicwI.jsx`)
- **React components**: PascalCase function names (e.g., `function GadkiStronaGwna()`)
- **CSS classes**: Inline Tailwind classes (not separate CSS files)
- **Variables**: camelCase
- **Routes**: kebab-case paths (e.g., `/dla-rodzicow-i-opiekunow`)

#### Assets

- Static files: Place in `./static` directory
- Images: Typically referenced from `c.animaapp.com` CDN (Anima pattern)
- For new assets, follow existing CDN pattern or place in `./static`
